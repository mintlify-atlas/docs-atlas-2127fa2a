A reliable toolchain is the foundation of every Python project. This page covers environment and version management, package installation, private repository hosting, building distributable artifacts, and reading configuration files.

## Environment Management

_Libraries for Python version and virtual environment management._

<CardGroup cols={2}>
  <Card title="pyenv" icon="code-branch" href="https://github.com/pyenv/pyenv">
    Simple Python version management.
  </Card>
  <Card title="pyenv-win" icon="windows" href="https://github.com/pyenv-win/pyenv-win">
    Pyenv for Windows.
  </Card>
  <Card title="uv" icon="rocket" href="https://github.com/astral-sh/uv">
    An extremely fast Python version, package and project manager, written in Rust.
  </Card>
  <Card title="virtualenv" icon="box" href="https://github.com/pypa/virtualenv">
    A tool to create isolated Python environments.
  </Card>

```bash
# pyenv — Python version management
pyenv install 3.12.0       # install Python version
pyenv global 3.12.0        # set global version
pyenv local 3.11.5         # set project-level version
```

```bash
# uv — fast Python package manager
uv init myproject          # create new project
uv add requests fastapi    # add dependencies
uv run python main.py      # run with managed env
uv sync                    # sync deps from pyproject.toml
```

## Configuration Files

_Libraries for storing and parsing configuration options._

<CardGroup cols={2}>
  <Card title="configparser" icon="sliders" href="https://docs.python.org/3/library/configparser.html">
    (Python standard library) INI file parser.

  <Card title="dynaconf" icon="gears" href="https://github.com/dynaconf/dynaconf">

  <Card title="hydra" icon="water" href="https://github.com/facebookresearch/hydra">
    Hydra is a framework for elegantly configuring complex applications.
  </Card>
  <Card title="python-decouple" icon="key" href="https://github.com/HBNetwork/python-decouple">
    Strict separation of settings from code.
  </Card>
  <Card title="python-dotenv" icon="file-lines" href="https://github.com/theskumar/python-dotenv">
    Reads key-value pairs from a `.env` file and sets them as environment variables.
  </Card>
</CardGroup>

```python
# python-dotenv — environment configuration
from dotenv import load_dotenv
import os

load_dotenv()  # loads .env file

DATABASE_URL = os.getenv("DATABASE_URL")
SECRET_KEY = os.getenv("SECRET_KEY", "default-dev-key")
```

```python
# dynaconf — advanced configuration
from dynaconf import Dynaconf

settings = Dynaconf(
    settings_files=["settings.toml", ".secrets.toml"],
    environments=True,
)

print(settings.DATABASE_URL)
print(settings.DEBUG)
```
